# Довідник флори та фауни

Індивідуальний проєкт із дисципліни "Веб-технології". Це веб-додаток для ознайомлення з біорізноманіттям (рослинний та тваринний світ), розроблений для навчального процесу.

## Про проєкт

Проєкт реалізовано на чистому **PHP** без використання сторонніх фреймворків (Laravel, Symfony тощо), за вимогою викладача. Для стилізації використано **Bootstrap 5** у мінімалістичному стилі.

### Основні функції:
- **Публічний каталог**: перегляд списку рослин та тварин.
- **Фільтрація**: сортування записів за категоріями (Флора/Фауна).
- **Детальні сторінки**: перегляд повної інформації про конкретний вид.
- **Адмін-панель**: захищений інтерфейс для керування контентом.
- **CRUD**: можливість додавання, редагування та видалення записів разом із зображеннями.

## Технологічний стек
- **Backend**: PHP 7.4+ (Vanilla PHP)
- **Database**: MySQL (InnoDB)
- **Frontend**: HTML5, CSS3, Bootstrap 5
- **Особливість**: **Zero Dependency** — проєкт не потребує сторонніх бібліотек або менеджера пакетів Composer, що забезпечує швидку роботу та легкий деплой на будь-який хостинг.

## Архітектура та реалізація

Проєкт побудований на архітектурному шаблоні **Micro-MVC**:
- **Core (Init/Functions)**: ядро системи, що відповідає за підключення до БД, завантаження конфігурації `.env` за допомогою власного парсера та загальні функції.
- **Controllers**: окремі файли в корені та папці `admin/`, які обробляють логіку запитів.
- **Views**: шаблони відображення, винесені в окрему директорію для чистоти коду.

### Безпека:
- **PDO Prepared Statements**: надійний захист від SQL-ін'єкцій.
- **CSRF Protection**: захист форм в адмін-панелі від міжсайтової підробки запитів.
- **Bcrypt Hashing**: безпечне зберігання паролів адміністраторів у базі даних.
- **Secure .htaccess**: обмеження прямого доступу до системних файлів та папки із зображеннями.

## Встановлення та деплой

### 1. Налаштування Бази Даних
1. Відкрийте панель керування БД (наприклад, phpMyAdmin).
2. Створіть нову базу даних або очистіть існуючу.
3. Імпортуйте структуру та початкові дані з файлу `database.sql`.

### 2. Конфігурація середовища
1. У корені проєкту знайдіть файл `.env`.
2. Вкажіть ваші дані для підключення до MySQL:
   ```env
   DB_HOST=ваш_хост
   DB_NAME=назва_бд
   DB_USER=користувач
   DB_PASS=пароль
   ```

### 3. Завантаження на хостинг (FileZilla)
1. Підключіться до вашого сервера через FTP-клієнт **FileZilla**.
2. Очистіть цільову папку (зазвичай `htdocs` або `public_html`).
3. Скопіюйте всі файли проєкту безпосередньо в цю папку. Проєкт автоматично почне працювати без додаткових команд.
